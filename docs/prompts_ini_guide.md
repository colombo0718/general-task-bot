# prompts.ini 撰寫規則

## 基本結構

```ini
[extractors]
<prompt_key> =
    <prompt 內容（多行縮排）>
```

- 只有一個 section：`[extractors]`
- 每個 key 對應一個「萃取器」，主程式用 `prompt_key` 引用它
- 多行內容用**縮排**（空格或 tab）延續，不須加引號
- `;` 開頭為 ini 格式的註解，不會被讀入

---

## 每個萃取器的標準段落結構

```
嚴格限制：只輸出最終答案，不得有多餘字元、前後綴或自然語言。
只允許：<允許的值>
【輸出格式（必須完全匹配其一）】<正規表示式>

任務：<說明這個萃取器在做什麼>
規則：
<條件 1> → <輸出值>
<條件 2> → <輸出值>
...
其他/無法判斷 → null（或預設值）
```

分為四個區塊：

| 區塊 | 作用 |
|------|------|
| 嚴格限制 | 告訴 LLM 輸出格式限制，防止多餘文字 |
| 輸出格式 | 用正規表示式精確約束，讓 LLM 自我校正 |
| 任務 | 一句話說明目的 |
| 規則 | 條件 → 輸出值的映射清單 |

---

## 輸出格式類型參考

| 用途 | 範例輸出格式 |
|------|-------------|
| 多選一 enum | `^(?:valueA\|valueB\|null)$` |
| 整數 | `^(?:[0-9]+\|null)$` |
| 有正負的整數 | `^-?\d+$` |
| 日期 YYYY-MM-DD | `^(?:\d{4}-\d{2}-\d{2}\|null)$` |
| 年月 YYYY/MM | `^(?:20[0-9]{2}/(?:0[1-9]\|1[0-2])\|null)$` |
| ISO 8601 含時區 | `^(?:20[0-9]{2}-...\+08:00\|null)$` |
| 中文或英數詞 | `^(?:[\u4e00-\u9fffA-Za-z0-9]{1,20}\|null)$` |
| 布林 0/1 | `^(?:0\|1)$` |

## 撰寫重點

### 1. 永遠要有 fallback
規則最後必須加上「無法判斷 → null」或預設值，避免 LLM 自行發明輸出。

### 2. 排除雜訊要明列
若有容易混淆的詞（例如日期數字 vs 金額數字），明確寫出排除規則。

```
- 排除看似日期（例：2023、7月、2024/05/01）、編號（例：股票代號）
```

### 3. 歧義處理要定義優先順序
同時出現多個候選值時，說明選哪個：

```
- 若同一句中提到多個候選，選擇最貼近「從」「起」「開始」語義的那一個
```

### 4. 需要日期計算時
說明以 `[REFERENCE].today_date` 為基準，並定義常見相對詞彙的換算方式：

```
- 今天/本日 → today_date
- 明天 → today_date + 1 天
- 昨天 → today_date - 1 天
- 下個月 → today_date 的月份 + 1
```

萃取器的 field 必須在 `mission.json` 設定 `"reference": true` 才會收到 today_date。

### 5. 加範例區塊
特別是欄位格式複雜或容易誤判時，加 `範例：` 區塊有助於 LLM 校準：

```
範例：
- 「幫我新增員工代號1001」→ 1001
- 「員工編號是 88」→ 88
- 「幫我加200元」→ null
```

### 6. 矛盾語意要明定勝負
當輸入同時包含相反語意時，指定哪方優先：

```
- 若同時出現矛盾（含稅 + 不含稅），以「不含稅」為優先 → N
```

## 系統保留 key（位於 `prompts_system.ini`）

這三個 key 被主程式硬編碼呼叫，**不可改名**，也不需要放在業務 `prompts.ini` 裡。

| key | 用途 | 輸出值 |
|-----|------|--------|
| `extracted_ronot` | 判斷使用者是否確認/取消待定指令 | `true` / `false` / `null` |
| `extract_run_at` | 萃取延後執行時間 | ISO 8601（`+08:00`）或 `null` |
| `extract_view_todos` | 判斷是否要查看待辦清單 | `true` / `false` |

## 完整範例

```ini
[extractors]
identify_expense =
    嚴格限制：只輸出最終答案，不得有多餘字元、前後綴或自然語言。
    只允許以下三選一（區分大小寫）：
    expenseRecord / queryTotal / null
    【輸出格式】^(?:expenseRecord|queryTotal|null)$

    用途：判斷輸入屬於「記錄支出」還是「查詢總額」。
    規則：
    1) 若輸入包含數字或金額描述（例如「午餐 120」「花了一百元」）→ expenseRecord
    2) 若輸入包含查詢字眼（花多少、總共多少、花費統計、本月消費）→ queryTotal
    3) 其他或無法判斷 → null

get_price =
    嚴格限制：只輸出阿拉伯數字，不得有單位或其他字元。
    輸出：正整數或 null
    【輸出格式】^(?:[0-9]+|null)$

    用途：擷取消費金額。
    規則：
    - 取輸入文字裡最像金額的數字，中文數字需先轉為阿拉伯數字（如「一百二十」→120）。
    - 排除看似日期（例：2023、7月）、編號（例：股票代號、發票號碼）。
    - 若沒有金額資訊 → null
```

## 多個 prompts.ini 的命名規則

主程式啟動時可帶參數決定讀哪組設定：

```
python main.py          → prompts.ini
python main.py pos      → prompts_pos.ini
python main.py store    → prompts_store.ini
```

- 後綴只允許英數與底線，其他字元會被自動移除
- `prompts_system.ini` 永遠都會被載入，不受參數影響
