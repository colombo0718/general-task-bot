# GTB 信心值欄位升級與 Benchmark 規劃

- 日期：2026-04-14
- 用途：會前快速對齊（昨日討論＋部分實作現況）

---

## 1. GTB 信心值欄位升級方向（已討論，部分實作）

### 目標

讓 `classify_tree` 與 `fields` 都能選擇輸出模式：

1. 單一值（維持舊行為）
2. 值 + 信心值（支援人工勘誤閉環）

### 設計原則

- 在欄位配置中使用 `with_confidence=true/false`
- 只有 `true` 才要求輸出信心值
- `false` 或未宣告時，維持舊格式（向下相容）

### 目前做法

- 先在 `gtb_dev.py` 驗證，不直接改主程式 `gtb.py`
- `prompts_system.ini` 已加入兩種格式提示：
  - `format_value_only`
  - `format_with_confidence`

## 2. Benchmark 兩個層次規劃（已啟動）

### 第一層：LLM 層 Benchmark（已在跑）

檔案：

- `llm_clients.py`
- `scripts/llm_benchmark.py`

目的：

- 批量比較不同 provider/model 的速度與格式穩定性

指標：

- `success`：有回應即可
- `correct`：回覆格式符合規範

現況：

- 已往平行化版本前進，降低整輪測試時間

### 第二層：GTB 任務層 Benchmark（規劃中）

目的：

- 不經 Line OA，直接驗證完整鏈路：
  - 意圖分類
  - fields 萃取
  - 任務參數是否正確

方向：

- 增加 GTB CLI/題本模式
- 可用題本批量測試常見口語輸入與預期欄位

## 3. 銜接到阿全（全葳小幫手）的下一步

要進到「功能列表 / 願望清單 / 錯題本」，前置條件是先具備可量化的對錯判斷能力。

### 建議順序

1. 先完成信心值輸出與格式可驗證

3. 再進入阿全專案功能治理：

### 核心原則

先可測，再可改，最後可學習。
